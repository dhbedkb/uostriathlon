Falls back to DEFAULT_ORDER below when absent, which matches the order this template has always rendered in. - tile.visibility: an object mapping each content type to one of three states — always / hidden_initially / hidden (Phase 3, #4). This replaced the old always/hover/click/hidden model: a tile no longer picks *how* hidden content is revealed (hover vs click), only *whether* it starts hidden. Whether a tile can be opened at all is a Section-level (or per-tile override) yes/no — see "Expandable" below — and if it can, the site's one global Interactions > "How hidden content is revealed" setting decides hover vs click for every expandable tile on the site. Badge and the numbered index badge are NOT part of content_order — they're absolutely-positioned overlay tags, not flow content, so reordering them has no visual meaning. They always render exactly as before.
