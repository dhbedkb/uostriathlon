{% comment %} Renders one Tile. A tile is: an optional Style (accent), an optional Badge (fixed corner tag, not flow content), and a single ordered list of Content Blocks — this IS the tile's content and its order. There is no separate content_order or visibility field anywhere — dragging a block in the editor's `blocks` list changes the render order directly, because it changes tile.blocks directly. Visibility works through one special block type: `expand`. Every block before the first `expand` block renders always-visible. Every block from `expand` onward renders inside the hover/click reveal region. No `expand` block present (or one with nothing after it) → the tile renders flat, no toggle, no
— everything shows. A second `expand` block later in the list is inert; only the first one means anything. Which trigger opens an expandable tile — hover or click — is one site-wide setting: site.data.settings.interactions.expand_trigger. There's no per-section or per-tile override any more: a tile either has something to reveal or it doesn't, and every tile that does reveals it the same way.
